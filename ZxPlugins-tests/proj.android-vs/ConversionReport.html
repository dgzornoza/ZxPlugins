General Information

VS Android Development

Message
Editing your Android project in Visual Studio will use Visual Studio's SDK and NDK.
Visual Studio 2015's build system currently supports only 32-bit x86 and ARM.
Ensure includes and preprocessor definitions have been converted correctly by checking them in your C++ project's NMake property page under IntelliSense | Include Search Path and IntelliSense | Preprocessor Definitions.
The packaging project build system does not support projects with more than one file with a .properties extension.

Troubleshooting

Message
If you are working in a pre-existing directory, clean all build output folders before trying to build with Visual Studio.
NDK Debugging is enabled by default for the Debug configuration only and necessary for deployment and C++ debugging. To disable it, remove NDK_DEBUG=1 from "Build Command Line" in your C++ project's NMake property page or switch to the Release configuration.
If you experience Ant not being able to start the JVM due to lack of heap space on packaging project build, close all other open programs and try again or increase the Java heap size using the -Xmx option.
Ensure relative file paths in the converted project and any configuration or .properties files are correct after conversion.
If your project.properties file references an external android library, make sure it is built already or contains a build.xml file.
By default, the .apk file generated by your Packaging Project will have the same name as the project. However, you cannot deploy an .apk with spaces in its filename to an Android device. If the package manager fails to install your app with an 'InvalidUri' code, remove the spaces from your .apk filename in the packaging project's General property page.
Make sure your C++ project is building libraries for the appropriate ABIs and platforms in the Application.mk file.
